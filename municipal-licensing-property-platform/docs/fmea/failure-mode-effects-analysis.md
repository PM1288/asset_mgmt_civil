# Failure Mode and Effects Analysis (FMEA)

| Component | Failure mode | Probable causes | Effect | Detection | Severity | Likelihood | Detectability | Mitigation / prevention | Automated recovery | Manual recovery | Degraded mode | Long-term risk | Signals |
|---|---|---|---|---|---|---|---|---|---|---|---|---|---|
| Caddy proxy | Proxy unavailable | config error, port conflict, container crash | Users cannot reach portal | container health, smoke tests | High | Medium | High | versioned config, health checks, simple routing | container restart | revert config, restart container | none | outage recurrence if config drift persists | restart count, 5xx |
| Frontend | SPA not served | broken build, nginx failure | UI unavailable, API may still be reachable | health check, browser error | Medium | Medium | High | immutable build image, smoke tests | container restart | redeploy previous image | API remains up | user support load | health endpoint, browser console |
| API | Request handling failure | code defect, resource exhaustion | core workflows fail | `/health/live`, logs, alert on 5xx | High | Medium | High | bounded resources, compile-tested code, input validation | restart policy | rollback release | readonly if only some endpoints affected | repeated instability | 5xx rate, latency |
| Keycloak | Auth outage | db outage, config issue, token signing issue | new logins fail | readiness, user reports | High | Medium | Medium | separate DB, JWT cache, realm export | container restart | restore auth DB, fix realm/client | existing cached-key token validation may continue temporarily | stale auth config, lockouts | login failures |
| Optional LDAP | Federation failure | network issue, wrong bind DN, directory outage | federated login fails | Keycloak admin logs | Medium | Medium | Medium | optional federation, local break-glass admin accounts | none | disable federation temporarily | local Keycloak users still work | stale identity sync | auth admin events |
| PostgreSQL app DB | unavailable | container crash, storage failure, corruption | reads/writes fail | readiness, connection errors | Critical | Medium | High | health checks, backups, isolation | container restart | restore from backup | none | data loss if backups invalid | `pg_isready`, app 503 |
| PostgreSQL growth | slow degradation | audit growth, document metadata growth | slower queries, backup duration increase | DB size monitoring | Medium | High | Medium | retention jobs, keep blobs out of DB, indexes | cleanup job | archive old data, reindex, tune vacuum | partial slowdowns | chronic performance decline | DB size, query latency |
| Redis | unavailable | memory pressure, crash, disk issue | rate limiting fallback, worker queue interruption | readiness, worker errors | High | Medium | High | memory cap, AOF, fallback logic | restart policy | restore service, inspect data dir | API continues with local throttle fallback; async jobs pause | backlog bursts after recovery | queue depth, redis ping |
| Celery worker | task backlog growth | redis issue, task failure, insufficient concurrency | backup/cleanup delays | queue depth, stale tasks | Medium | Medium | Medium | low prefetch, separate worker, task time limits | restart worker | scale worker, clear poison tasks | foreground APIs continue | missed maintenance windows | task age, queue length |
| Celery beat | scheduler stalled | crash, misconfiguration | backups/cleanup not triggered | absence of expected artifacts | High | Low | Medium | dedicated beat service, logs, manual scripts | restart beat | trigger scripts manually | system still usable but maintenance deferred | backup gap | missing backups |
| Document store | file write/read failure | disk full, permission issue, corruption | document upload/download impacted | diagnostics, audit failures | High | Medium | High | encrypted local storage, disk thresholds | none | expand disk, restore volume | record workflows continue without uploads | evidence/document gap | free space, I/O errors |
| Backup subsystem | backup chain corruption | partial writes, checksum mismatch | restore confidence lost | validation script | Critical | Medium | High | checksum manifest, scheduled validation | weekly validation task | create fresh validated backup, investigate storage | none | hidden DR weakness | validation failure |
| Restore workflow | restore fails | version mismatch, operator error, corrupt dump | delayed recovery | restore exercise results | Critical | Medium | Medium | tested scripts, `pg_restore --list`, runbooks | none | iterative recovery with runbook | prolonged outage | untested recovery path | restore logs |
| Log storage | runaway growth | verbose logging, repeated failures | disk pressure | disk metrics, host alerts | High | Medium | High | rotation, Docker log caps, structured logs | rotation | purge old logs after backup | core service may continue briefly | sudden host outage | disk usage |
| Container restart loop | repeated crash | invalid config, failed migration, missing secret | service unavailable | restart count, docker ps | High | Medium | High | startup preflight, config validation | restart policy only | fix config and redeploy | none | alert fatigue | restart count |
| Memory leak style degradation | slow heap growth | library bug, unbounded caches | latency and crash over days/weeks | host metrics, container RSS | Medium | Medium | Medium | bounded caches, process restarts on deploy, memory limits | OOM restart | investigate logs, patch and redeploy | temporary service loss | recurring instability | RSS trend |
| Flapping network | intermittent dependency access | host network issue, DNS instability | spiky errors and retries | latency spikes, auth/redis errors | Medium | High | Medium | retry budget, jitter, circuit breaker, fallback | retries with budget | stabilize network, inspect DNS | limited degraded service | hidden operator toil | timeout counts |
| Certificate expiry | expired enterprise cert or mismanaged internal CA trust | missed renewal | TLS failures | smoke tests, browser warnings | High | Low | High | internal CA defaults, documented replacement path | none | replace certs / trust chain | none | complete edge outage | TLS handshake failures |
| Host reboot / power loss | abrupt stop | power event, maintenance reboot | temporary outage, possible crash recovery | Windows event log, container restarts | High | Medium | High | durable volumes, restart policies, AOF, PostgreSQL WAL | containers restart automatically | verify integrity, run smoke tests | brief outage | repeated host instability | boot event timestamps |
